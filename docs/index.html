GitHub Copilot, right next to your tabs.

Anya brings GitHub Copilot into your browser sidebar — the same agent you use in your editor, now sitting next to the pages you're actually working in. Tell it what you want done — fill a form, click through a wizard, dig across tabs, or tidy your bookmarks — and optionally point it at a local project folder so it can reason over your code while driving your tabs. Signs in with your GitHub Copilot account, runs on your laptop, drives your real logged-in browser.

Install — one line

$ curl -fsSL https://raw.githubusercontent.com/aasis21/Anya/main/install.sh | bash

01 · Why you'd actually use it

Your editor's Copilot,
but for the other half of your day.

Half your work isn't in the IDE — it's in PRs, dashboards, tickets, docs, forms, and twenty open tabs. Anya is the same Copilot, in that half.

  1. 01

    Stop reading things twice.

    That 600-line PR, the 40-page RFC, the noisy ICM thread. @tab tldr and you're done. Anya reads what's on screen — no copy-paste, no second tab, no tldr-bot.

  2. 02

    Skip the click-through tax.

    Repetitive forms, multi-step wizards, "find the right button in the new ADO UI." Describe the outcome; Anya drives your logged-in browser to get there. No reauth, no captchas you have to redo.

  3. 03

    Make 20 tabs answer one question.

    "Does the spec match what we shipped?" "Which of these dashboards is red?" @tabs hands Anya every open tab as context. Stop alt-tabbing — start asking.

  4. 04

    Bring your repo into the loop.

    Point Anya at a local project folder and it gets the SDK's full code toolbelt — read files, grep, edit, run shells, MCP. Reason over your code while driving the tab that depends on it.

  5. 05

    Treat bookmarks like a knowledge graph.

    "Open the order release page." "Group AI / infra / fun." Anya searches your bookmarks before guessing URLs, and reorganises in bulk after you OK the plan. Your saved stuff finally pays rent.

  6. 06

    Yours. Stays yours.

    Signs in with your existing GitHub Copilot account — same seat, same model. Chats live in your browser. The bridge runs on your laptop. Nothing leaves the machine.

02 · How you talk to it

Three little prefixes.
That's the whole language.

Just type — nothing special needed. But when you want to be precise, three characters give Anya everything it needs.

@

context

"Look at what I'm looking at."

Pulls in pieces of your browser — automatically inlined before sending.

  • @tab   this page
  • @selection   what I've highlighted
  • @tabs   everything open
  • @clipboard   what I last copied
/

commands

"Do something to the sidebar."

Quick actions on your chat — never reaches the model.

  • /new   start a fresh chat
  • /pin · /tag add ux
  • /search ordering
  • /export   save as Markdown
#

references

"Look up the thing I labelled X."

Named lookups — bookmarks, prior chats, saved snippets.

  • #bookmark/…
  • #chat/…
  • — coming soon —

One line. Any Chromium.

Then, in your browser

  1. Open edge://extensions (or your browser's equivalent)
  2. Toggle Developer mode
  3. Load unpacked → pick ~/Anya/extension/dist/
  4. Pin the action icon → click → sidebar opens
  5. Type ping → expect PONG

Options

-InstallDir <path>where to clone (default ~/Anya)
-Branch <name>track a non-main branch
-Browsers edge,chromeregister only specific browsers
--install-dir <path>where to clone (default ~/Anya)

04 · Under the hood

Two small pieces.
That's the whole thing.

A sidebar extension talks to a tiny Node bridge on your laptop. The bridge wraps the official @github/copilot-sdk and shells out to Playwright when Anya needs to drive a tab. No servers, no relays, no telemetry.

┌────────────────────┐    JSON frames    ┌────────────────────┐
│  Browser sidebar   │ ◀─────────────▶   │  Node bridge       │
│  (Lit + marked)    │  Native Messaging │  @github/copilot-  │
│                    │                   │  sdk + tools       │
└────────────────────┘                   └────────────────────┘
        │                                          │
        │ chrome.tabs / scripting                  │ shells out to
        ▼                                          ▼
   browser context                          playwright-cli
