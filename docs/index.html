01 · Why you'd actually use it

Your editor's Copilot,
but for the other half of your day.

Half your work isn't in the IDE — it's in PRs, dashboards, tickets, docs, forms, and twenty open tabs. Anya is the same Copilot, in that half.

  1. 01

    Sees your browser.

    Ask about any page and Anya reads it — tabs, bookmarks, history, selections. Need precision? Right-click → "Add to Anya" to pin a specific section, table, or field. Or use the 📎 menu and @ references.

  2. 02

    Acts on your browser.

    Fill forms, click through wizards, navigate dashboards — Anya drives your logged-in browser via Playwright. Focus a text field and ask what to write — one click inserts it.

  3. 03

    Your repo too.

    Attach a local folder — the SDK loads your skills, prompts, and file tools. Read files, grep, edit, run shell commands, all while the AI is also looking at your browser.

  4. 04

    Nothing leaves your machine.

    Your existing GitHub Copilot seat, your browser, your laptop. Chats live in chrome.storage. The bridge runs locally. No server, no telemetry, no cloud sync.

One command.
Every Chromium browser.

Then, in your browser

  1. Open edge://extensions (or your browser's equivalent)
  2. Toggle Developer mode
  3. Load unpacked → pick ~/Anya/extension/dist/
  4. Pin the action icon → click → sidebar opens
  5. Start chatting — right-click anything to add context

To update, run the same command again. See the README for options, uninstalling, and developing locally.

04 · Under the hood

Simple architecture.
Serious power.

A sidebar extension, a content script on every page, and a local Node bridge. No servers. No relays. Everything on your machine.

Sidebar chat UI · context chips · 📎 menu
◀ ─── JSON frames ─── ▶ Native Messaging
Node bridge @github/copilot-sdk · browser tools
↑ right-click · Alt+A · field tracking Page bridge
↓ click · type · screenshot · navigate playwright-cli

The page bridge runs as a content script on every page — captures elements you right-click, tracks text fields, and sends context to the sidebar. The Node bridge wraps the Copilot SDK and runs tools. That's it.
